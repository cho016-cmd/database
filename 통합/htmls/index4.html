연구자: 김강현

1. 연구 주제

공공데이터 API를 활용한 실시간 버스 도착 알림 시스템 개발

2. 서론

가. 연구 동기 및 필요성

매일 버스를 이용하지만, 버스 도착 시간은 예상과 다른 경우가 많습니다. 어떤 날은 너무 빨리 와서 놓치고, 어떤 날은 너무 늦게 와서 하염없이 기다리기도 합니다. 스마트폰 앱으로 도착 정보를 확인할 수 있지만, 계속 화면을 확인해야 하는 불편함이 있고, 잠시 한눈을 팔면 버스를 놓치기 일쑤입니다. 이런 불편한 점을 해결하기 위해서 버스 정보를 실시간으로 불러오면서, 버스가 곧 도착할 것 같을 때 알림이 울리는 것이 더 편리할 것 같다는 생각에 이 연구를 시작하게 되었습니다.

단순히 정보를 보여주는 것을 넘어, 사용자에게 필요한 정보를 적절한 때에 '능동적으로' 알려주는 시스템을 개발함으로써 바쁜 현대인의 시간 낭비를 줄이고 대중교통 이용 경험을 긍정적으로 바꾸는 데 기여하고자 합니다.

나. 연구 목적

본 연구는 공공데이터 API를 통해 수집한 실시간 버스 위치 정보를 활용하여, 사용자가 지정한 버스가 정류장에 도착하기 전에 미리 알려주는 웹 기반의 실시간 알림 시스템을 개발하는 것을 최종 목적으로 합니다.

다. 연구 문제

  • 공공데이터 API를 통해 어떻게 실시간 버스 도착 정보를 안정적으로 수신할 수 있는가?
  • 사용자에게 가장 효과적인 버스 도착 알림 방식(시각적, 청각적)은 무엇이며, 이를 웹 환경에서 어떻게 구현할 것인가?
  • 사용자 편의성을 고려한 최적의 알림 시점(예: 도착 3분 전)을 결정하고 시스템에 적용하는 방안은 무엇인가?

3. 이론적 배경

가. 관련 이론/기술 및 선행 연구 조사

공공데이터 API (REST API)

공공데이터포털에서 제공하는 '버스도착정보조회 서비스' API를 활용합니다. 정해진 형식으로 요청을 보내면 실시간 버스 운행 정보를 응답으로 받아오는 기술을 이해하고 활용합니다.

웹 프론트엔드 (HTML/CSS/JavaScript)

사용자가 버스 노선과 정류장을 선택하고 알림을 설정할 수 있는 화면을 구축합니다. JavaScript를 이용해 주기적으로 API를 호출하고, 응답받은 데이터를 화면에 표시하며, 조건에 맞을 때 알림을 발생시키는 로직을 구현합니다.

웹 푸시 알림 (Web Push Notifications)

브라우저가 백그라운드 상태에 있거나 꺼져있을 때도 사용자에게 알림을 보낼 수 있는 기술입니다. 서비스 워커(Service Worker)를 사용하여 서버로부터 푸시 이벤트를 받아 사용자에게 알림을 표시하는 방법을 탐구합니다.

4. 연구 내용 및 방법

가. 연구 개발 대상

  • 대상: 특정 버스 노선 및 정류장을 이용하는 사용자
  • 플랫폼: 웹 브라우저 기반의 알림 서비스 페이지

나. 연구 절차 및 방법

  1. 요구사항 분석 및 설계: 알림 시스템에 필요한 핵심 기능(노선/정류장 선택, 알림 시간 설정 등)을 정의하고 사용자 인터페이스(UI)를 설계합니다.
  2. 개발 환경 구축: 개발에 필요한 도구(코드 에디터, 웹 서버 등)를 설치하고 공공데이터 API 활용을 신청합니다.
  3. 핵심 기능 개발: JavaScript를 이용해 주기적으로 API를 호출하여 데이터를 가져오고, 지정된 조건(예: 3분 내 도착)이 되면 웹 알림을 발생시키는 로직을 구현합니다.
  4. 사용자 테스트 및 개선: 완성된 프로토타입을 직접 사용해보면서 불편한 점이나 버그를 찾아 수정하고 기능을 개선합니다.

5. 기대 효과 및 활용 방안

본 연구를 통해 개발된 실시간 버스 도착 알림 시스템은 사용자가 더 이상 스마트폰 화면을 계속 주시할 필요 없이, 다른 활동을 하다가 버스 도착 시간에 맞춰 준비할 수 있도록 도와줍니다. 🚌

이를 통해 바쁜 학생과 직장인들의 시간 낭비를 줄여주고, 버스를 놓칠지 모른다는 불안감을 해소하여 대중교통 이용 경험의 질을 크게 향상시킬 수 있습니다.

궁극적으로는 이러한 편리한 시스템이 대중교통 이용을 활성화하는 데 긍정적인 영향을 미쳐, 교통 혼잡 완화 및 환경 보호와 같은 사회적 가치 창출에도 기여할 수 있을 것으로 기대됩니다. ✨
